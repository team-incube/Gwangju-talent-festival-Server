# Agent Flywheel

## 개념

Flywheel은 각 개발 사이클이 다음 사이클을 개선하는 자기강화 루프다.
처음에는 돌리기 어렵지만, 회전이 쌓일수록 더 쉽게 더 빠르게 돌아간다.

## 사이클 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│  요청                                                             │
│   ↓                                                              │
│  [/planning] ─── learnings.md + patterns.md 참조               │
│   ↓                                                              │
│  [feature 에이전트] ── 코드 작성                                  │
│   ↓                                                              │
│  [test 에이전트] ──── 테스트 작성                                 │
│   ↓                                                              │
│  [review 에이전트] ── 코드 리뷰                                   │
│   ↓                    ↓                                         │
│  APPROVE        REQUEST CHANGES                                  │
│   ↓                    ↓                                         │
│  [pr 에이전트]   [bugfix|refactor 에이전트]                       │
│   ↓                    ↓                                         │
│  PR 생성        수정 후 [review 에이전트]                         │
│   ↓                    ↓                                         │
│  PR 피드백      APPROVE → [pr 에이전트]                           │
│   ↓                                                              │
│  [feedback 에이전트] ── 피드백 라우팅                              │
│   ↓                                                              │
│  learnings.md 업데이트 ←── 핵심! 학습 기록                        │
│   ↑_______________↓                                              │
│          다음 사이클에서 더 나은 코드                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 파일 구조

| 파일 | 역할 |
|------|------|
| `learnings.md` | 과거 실수와 개선사항 기록 |
| `patterns.md` | 반복 패턴과 안티패턴 |
| `sessions.md` | 자동 기록되는 세션별 수정 파일 목록 |
| `session.log` | 현재 세션 임시 로그 (stop 훅이 sessions.md로 병합) |

## Flywheel 효과 측정

- PR 당 평균 리뷰 코멘트 수 추이
- 반복되는 피드백 패턴 감소 여부
- 빌드 실패 빈도
- 테스트 커버리지 증가 추이
